### 학습 목표

- 좋은 프롬프트와 나쁜 프롬프트 구분
- 구체적이고 명확한 프롬프트 작성
- 반복적 개선 기법

### 프롬프트 엔지니어링 원칙

> [!NOTE]
>
> **프롬프트**는 AI 에이전트와 협업하기 위해 사용되는 텍스트 기반의 지시/설명/질문을 의미합니다. AI 에이전트가 최상의 결과물을 생성할 수 있도록 더 효과적인 형식, 구문, 단어, 기호를 선택하는 작업을 **프롬프트 엔지니어링**이라고 합니다. ([더 알아보기](https://aws.amazon.com/ko/what-is/prompt-engineering))

**DON'T (하지 말아야 할 것)**
- ❌ 불명확한 요구사항
- ❌ 기술 스택과 제약사항 미지정
- ❌ 너무 많은 기능을 한 번에 요청

**나쁜 프롬프트 예시**

```md
웹사이트 만들어 줘
```

**DO (해야 할 것)**
- ✅ 항목별/단계별로 요구사항 세분화
- ✅ 기술 스택과 제약사항 지정
- ✅ 예상 결과물 설명

**좋은 프롬프트 예시**

```md
HTML, CSS, JavaScript를 사용해서 할 일 관리 웹 애플리케이션을 만들어 주세요.

기능 요구사항:
- 할 일 추가/삭제/완료 표시
- 로컬 스토리지에 데이터 저장
- 반응형 디자인

기술 스택:
- 바닐라 JavaScript 사용 (프레임워크 없음)
- 모던 CSS (Flexbox/Grid 활용)
- 접근성 고려

제약사항:
- 외부 API 사용 금지

예상 결과물:
- index.html
- style.css
- script.js
```

### Vibe 세션으로 To-Do 웹 사이트 제작하기

1. Git Bash 등의 터미널 환경에서 새 프로젝트 폴더 "todo-app"을 생성합니다. kiro 명령어로 프로젝트 폴더를 엽니다.

> [!NOTE]
>
> Do you trust the authors of the files in this folder? 경고 창이 나타나면 "Yes, I trust the authors"를 선택합니다.

```sh copy
mkdir todo-app
cd todo-app
kiro ./
```

2. "todo-app" 프로젝트 창을 열고 아래 프롬프트를 복사한 후, Vibe 세션에 입력하여 기본 구조를 요청합니다. ([예시](https://youtu.be/-IjYrIwRn54?si=3uqR9r0H2ZO8Jjwo&t=100))
```md copy
할 일 관리 앱의 기본 HTML 구조를 만들어 주세요.

요구사항:
- 제목: "My Todo App"
- 할 일 입력 필드와 추가 버튼
- 할 일 목록을 표시할 영역
- 깔끔하고 현대적인 디자인
- 모바일 친화적인 레이아웃

파일명: index.html
```

3. 아래 프롬프트를 복사한 후, Vibe 세션에 입력하여 스타일링을 요청합니다. ([예시](https://youtu.be/-IjYrIwRn54?si=HIro2r2GqScbsCLS&t=140))
```md copy
방금 만든 HTML에 맞는 CSS 스타일을 작성해 주세요.

디자인 요구사항:
- 중앙 정렬된 카드 형태의 레이아웃
- 부드러운 그림자와 둥근 모서리
- 색상 테마: 파란색 계열
- 버튼 호버 효과
- 반응형 디자인 (모바일 대응)

파일명: style.css
```

> [!NOTE]
>
> Kiro가 생성한 **index.html** 파일을 열고, 인터넷 브라우저에서 스타일링이 적용되었는지 확인합니다. 
> 
> ([예시](https://youtu.be/-IjYrIwRn54?si=QiZIj-vpXoexQlFh&t=130))

4. 아래 프롬프트를 복사한 후, Vibe 세션에 입력하여 스타일링을 개선합니다. ([예시](https://youtu.be/-IjYrIwRn54?si=_e1x-7I91HJ2CUrT&t=160))
```md copy
CSS를 다음과 같이 개선해 주세요:
- 할 일 항목에 체크박스 스타일 개선
- 완료된 항목은 취소선과 흐린 색상 적용
- 삭제 버튼을 빨강색으로 변경
- 전체적인 간격 조정
```

> [!NOTE]
>
> 🔄 **반복 개선** : **index.html** 파일을 열고, 인터넷 브라우저에서 새로고침한 후 변경사항을 확인합니다. 스타일링 결과가 마음에 들지 않으면 구체적인 피드백을 주고 작업을 다시 요청하세요.

5. 아래 프롬프트를 복사한 후, Vibe 세션에 입력하여 주요 기능을 추가합니다. ([예시](https://youtu.be/-IjYrIwRn54?si=RXj7diWm_L1xDERD&t=185))

> [!NOTE]
>
> 💡 **컨텍스트** : `#file`을 통해 특정 파일을 참조할 수 있으며, `#file`을 입력하고 Tab/Enter를 입력하면 파일 선택 창이 나타납니다.

> [!NOTE]
>
> 💡 **컨텍스트** : `#folder`를 통해서는 프로젝트 내 특정 디렉터리(폴더)를 참조할 수 있습니다. `#folder`를 입력하고 Tab/Enter를 입력하면 디렉터리(폴더) 선택 창이 나타납니다.

```md copy
#index.html #style.css

기존 HTML과 CSS에 맞는 JavaScript 코드를 작성해 주세요.

기능:
- 할 일 추가 기능
- 할 일 삭제 기능  
- 완료 상태 토글 기능
- 로컬 스토리지에 데이터 저장하기/불러오기

파일명: script.js
```

> [!NOTE]
>
> 🔄 **반복 개선** : **index.html** 파일을 열고, 인터넷 브라우저에서 새로고침한 후 동작을 테스트합니다. 오류가 발생하면 구체적인 피드백을 주고 다시 요청하세요. ([예시](https://www.youtube.com/watch?v=-IjYrIwRn54&t=215s))

6. 아래 프롬프트를 복사한 후, Vibe 세션에 입력하여 프로젝트 개선 방안을 요청합니다. ([예시](https://www.youtube.com/watch?v=-IjYrIwRn54&t=228))

```md copy 
현재 프로젝트의 전체 구조를 분석하고, 
코드 품질을 개선할 수 있는 방법을 제안해 주세요.
```
