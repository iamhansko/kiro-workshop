[Kiro](https://github.com/kirodotdev/Kiro)는 AWS에서 개발한 AI 통합 개발 환경(IDE)으로 2025년 11월 18일 정식 출시되었습니다. 주요 특징은 다음과 같습니다.

- **AI 네이티브** : 

    - Kiro는 처음부터 AI와의 협업을 염두에 두고 설계된 개발 도구입니다. 스펙 세션 / 스티어링 / 에이전트 훅 / MCP 서버 등 AI에 특화된 다양한 기능을 기본 제공합니다.

- **컨텍스트 유지** : 

    - Kiro는 프로젝트 정보(컨텍스트)를 지속적으로 기억하고 작업에 자동 반영합니다. 이 덕분에 Kiro에게 똑같은 상황 설명을 두 번 반복할 필요가 없습니다.

- **구조화된 개발** : 

    - ( *Spec Driven Development* ) 개발자의 설명과 지시를 구조화된 Spec 문서로 변환할 수 있으며, 이 경우 Kiro의 모든 계획, 설계, 개발 작업이 Spec 문서를 기준으로 진행됩니다. 모든 작업이 문서에 따라 진행되기 때문에 Kiro의 활동을 추적하기 쉽고, Kiro의 잘못된 이해나 실수를 찾아 오류를 줄일 수 있습니다.

- **자동화 지원** : 

    - 에이전트 훅을 통해 개발자의 반복 작업을 자동화할 수 있습니다. 번거로운 반복 작업은 Kiro가 개발자를 대신하여 자동 수행하기 때문에 작업 시간을 줄일 수 있습니다. 

- **확장 지원** : 

    - MCP를 통해 외부 도구를 연동할 경우 Kiro가 더 많은 정보를 수집하고, 더 많은 기능을 활용할 수 있습니다. 예를 들어, [GitHub 도구](https://kiro.dev/docs/mcp/usage/#github-mcp-server)와 연동하면 Kiro는 코드 저장소에서 예시 코드를 확인하고, 더 높은 품질의 코드를 작성할 수 있습니다.

> [!NOTE]
> 
> ***Kiro***는 일본어로 "길/경로(きろ)"를 의미하며, 개발자의 Idea를 Production으로 이끄는 여정을 가리킵니다.

##### 🚀 Vibe 세션 (대화형 개발)

- 빠른 질문과 설명
- 대화를 통해 프로젝트 구축
- 대화에 따른 연속적인 변화를 계속 파악하며 탐색적 코딩

##### 📋 Spec 세션 (구조화된 개발)

- 요구사항 → 설계 → 구현 순서로 프로젝트 구축
- 문서 기반의 체계적인 개발 프로세스
- 작업 단계를 세분화하여 철저한 기능 구현 보장

##### ⚙️ Steering (개발 표준)

- 팀 코딩 스타일/규칙 자동 적용
- 일관된 코드 품질 유지

##### 🔧 Agent Hooks (자동화)

- 반복 작업 자동 수행
- 코드 품질 자동 검사

##### 🔗 MCP Servers (외부 도구 연동)

- 실시간 웹 정보 접근
- 외부 도구 통합
